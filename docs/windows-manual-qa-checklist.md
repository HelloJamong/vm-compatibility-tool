# Windows 수동 QA 체크리스트

## 1. 범위

이 문서는 Tauri 기반 VM Compatibility Tool의 수동 검증 체크리스트입니다.  
특히 다음 변경 이후 동작을 확인하는 데 사용합니다.

- 레지스트리 manifest 기반 점검 / 조치 구조
- GUI 재구성
- 선택형 추가 레지스트리 조치
- 시작 점검 화면 및 비활성화 흐름

## 2. 사전 조건

- Windows 10 / 11 환경에서 테스트합니다.
- 반드시 **관리자 권한**으로 실행합니다.
- 가능하면 **VM 또는 스냅샷 복구가 가능한 환경**을 사용합니다.
- 기준 테스트 머신 1대 이상은 아래 상태를 포함하도록 준비합니다.
  - Hyper-V / WSL / VBS / HVCI가 일부 또는 전부 활성화된 상태
  - 일부 레지스트리 값은 존재하고 활성 상태
  - 일부 레지스트리 값은 의도적으로 제거된 상태

## 3. 빌드 / 실행

- [ ] 앱을 관리자 권한으로 실행할 수 있다.
- [ ] release 빌드 실행 시 콘솔 창 없이 정상 실행된다.
- [ ] 앱 제목과 아이콘이 올바르게 표시된다.
- [ ] 관리자 권한 없이 실행하면 경고 후 종료된다.
- [ ] 배포 EXE 실행 시 현재 내부 배포 정책에 맞는 SmartScreen 동작이 나타난다.
- [ ] SmartScreen이 표시될 경우 `추가 정보 → 실행` 안내 문구가 실제 흐름과 일치한다.

## 4. 메뉴 / 내비게이션

- [ ] 메인 메뉴에 3개의 주요 작업 버튼이 표시된다.
- [ ] 가상화 점검 후 메뉴 배지가 정상적으로 갱신된다.
- [ ] 각 패널에서 뒤로 가기 시 메인 메뉴로 복귀한다.

## 5. 시스템 정보 화면

- [ ] `시스템 사양 체크`가 정상적으로 로드된다.
- [ ] OS / CPU / 메모리 / 디스크 / 부팅 / 메인보드 / GPU / 전원 정보가 표시된다.
- [ ] 이벤트 로그 요약에 한글 깨짐이 없다.
- [ ] 시스템 정보 CSV 내보내기가 성공하고 Excel / 메모장에서 정상적으로 열린다.

## 6. 가상화 점검

- [ ] `가상화 설정 점검`이 정상 완료된다.
- [ ] 요약 카드에 아래 집계가 올바르게 표시된다.
  - 조치 필요
  - 정상 / 비활성
  - 확인 불가
- [ ] Hyper-V / WSL / `hypervisorlaunchtype` 상태가 실제 시스템 상태와 일치한다.
- [ ] VBS / HVCI / Credential Guard / LSA 관련 레지스트리 항목이 예상대로 표시된다.
- [ ] Excluded legacy 항목은 **참고용 / 수동 선택 가능 항목**으로 취급된다.
- [ ] 참고용 항목이 자동 조치 대상처럼 오해되지 않는다.

## 7. 누락 레지스트리 값 정책

하나 이상의 manifest 대상 레지스트리 값을 테스트 전에 완전히 삭제한 뒤 검증합니다.

- [ ] 점검 결과에서 해당 값이 `미설정` 또는 이에 준하는 상태로 표시된다.
- [ ] 값이 없다는 이유만으로 해당 액션 그룹이 자동 선택되지 않는다.
- [ ] 비활성화 실행 후 로그에 **값이 없어서 생성하지 않았다**는 의미가 남는다.
- [ ] 재점검 시 Windows가 별도로 재생성하지 않은 이상 여전히 값이 없는 상태로 유지된다.

## 8. 선택적 비활성화 정책

일부 값은 활성 상태, 일부 값은 비활성 또는 누락 상태인 테스트 머신으로 검증합니다.

- [ ] 점검 결과에서 **실제로 존재하고 활성 상태인 값만** 조치 필요로 표시된다.
- [ ] 비활성화 미리보기 / 모달에서 기대한 그룹만 선택된다.
- [ ] 비활성화 실행 시 활성 상태였던 기존 값만 `0`으로 변경된다.
- [ ] 이미 비활성 상태인 값은 `이미 비활성 상태`로 기록된다.
- [ ] 누락된 값은 건너뛰고 생성하지 않는다.
- [ ] 재점검 결과가 변경된 비활성 상태를 반영한다.

## 9. Hyper-V / WSL / BCD

- [ ] Hyper-V 기능 감지가 정확하다.
- [ ] WSL / VirtualMachinePlatform 감지가 정확하다.
- [ ] `hypervisorlaunchtype`가 활성 상태면 조치 필요로 표시된다.
- [ ] 필요 시 비활성화 실행이 `hypervisorlaunchtype off`로 반영된다.
- [ ] 재점검 시 예상한 BCD 상태가 표시된다.

## 10. 추가 선택형 레지스트리 조치

- [ ] 기본 자동 조치 외에 수동 선택 가능한 추가 레지스트리 후보가 필요 시 표시된다.
- [ ] 체크하지 않은 추가 항목은 실행되지 않는다.
- [ ] 체크한 추가 항목만 실행 결과와 로그에 반영된다.
- [ ] `InspectOnly` 항목은 체크박스 대상이 아니다.
- [ ] Windows Hello for Business 항목은 경고 / 선행 확인 용도로만 표시된다.

## 11. 비활성화 패널 / 로그

- [ ] 선택된 조치 요약이 점검 결과와 일치한다.
- [ ] 진행 로그가 단계별로 표시된다.
- [ ] 성공 / 경고 / 건너뜀 / 누락값 로그가 시각적으로 구분된다.
- [ ] 실제 조치가 필요 없는 경우 “조치할 항목 없음” 의미가 명확히 보인다.

## 12. 재부팅 흐름

- [ ] 비활성화 성공 후 재부팅 버튼이 표시된다.
- [ ] 재부팅 버튼을 누르면 인앱 확인 대화상자가 열린다.
- [ ] 취소 시 부작용 없이 대화상자만 닫힌다.
- [ ] 확인 시 재부팅 예약이 정상 동작한다.

## 13. 시작 점검 화면 / 레이아웃

- [ ] 시작 점검 완료 후 안내 문구가 2줄로 자연스럽게 보인다.
- [ ] 완료 화면에서 footer 버전 정보가 스크롤 없이 보인다.
- [ ] 완료 화면의 요약 카드 / 경고 / 버튼 / footer가 한 화면 안에 정상 배치된다.
- [ ] 최소 지원 창 크기에서 버튼과 안내 문구가 겹치지 않는다.

## 14. 회귀 점검

- [ ] `npm run check`
- [ ] `cargo check --manifest-path src-tauri/Cargo.toml --target x86_64-pc-windows-msvc`
- [ ] 1100x750 이상 해상도에서 명백한 레이아웃 깨짐이 없다.
- [ ] 최소 지원 크기에서 주요 흐름이 사용 가능하다.

## 15. QA 중 기록할 항목

- Windows 버전 / 빌드
- SmartScreen 노출 여부와 실제 버튼 / 문구
- 초기 활성화되어 있던 보안 기능
- 존재했던 레지스트리 값과 누락된 레지스트리 값
- 참고용 / 수동 선택 항목이 실제로 유용했는지 여부
- 표현이 어색한 행 라벨 / 상세 문구 / 경고 문구
