# Windows 수동 QA 체크리스트

이 문서는 **현재 구현된 VM Compatibility Tool** 기준의 Windows 실기 검증 체크리스트입니다.
현재 앱의 실제 흐름(자동 시작 점검, CSV 자동 저장, 선택형 추가 레지스트리 조치, WHfB 경고, 전체화면 조치 모달)을 기준으로 정리했습니다.

---

## 1. 테스트 목적

아래 기능이 현재 구현 의도대로 동작하는지 확인합니다.

- 앱 시작 시 자동 점검 및 CSV 자동 저장
- 시스템 정보 / 가상화 점검 패널
- 선택형 비활성화 실행 (전체화면 조치 모달)
- 추가 선택형 레지스트리 조치
- Windows Hello for Business(WHfB) 경고 흐름
- 재부팅 유도 / 로그 / 백업 저장
- 고정 창 크기 레이아웃

---

## 2. 테스트 환경 준비

### 공통 조건
- [ ] Windows 10 또는 Windows 11 환경이다.
- [ ] 앱을 **관리자 권한**으로 실행할 수 있다.
- [ ] 가능하면 VM 또는 스냅샷 복구 가능한 테스트 장비를 사용한다.
- [ ] WebView2 사용 가능 환경이다.

### 권장 테스트 머신 상태
아래 상태를 모두 한 대에서 만들기 어렵다면, 2대 이상으로 나누어 확인합니다.

- [ ] Hyper-V 기능이 일부 또는 전부 활성화된 머신
- [ ] WSL / VirtualMachinePlatform이 활성화된 머신
- [ ] VBS / HVCI / Credential Guard / LSA 관련 값이 일부 활성화된 머신
- [ ] 일부 대상 레지스트리 값이 아예 없는 머신
- [ ] WHfB가 활성화된 머신 또는 비활성 머신

---

## 3. 실행 / 배포 기본 확인

- [ ] release EXE를 관리자 권한으로 실행할 수 있다.
- [ ] release 빌드에서 콘솔 창 없이 앱이 열린다.
- [ ] 앱 제목 / 아이콘이 정상적으로 표시된다.
- [ ] 관리자 권한 없이 실행하면 경고 후 종료된다.
- [ ] 내부 배포 EXE 실행 시 현재 배포 정책에 맞는 SmartScreen 흐름이 나타난다.
- [ ] SmartScreen이 나타나면 문서/릴리즈 노트의 `추가 정보 → 실행` 안내와 실제 흐름이 일치한다.

---

## 4. 시작 자동 점검 흐름

앱 실행 직후 표시되는 **시작 점검 화면(InspectionSummaryModal)** 기준입니다.

- [ ] 앱 시작 직후 별도 조작 없이 자동 점검이 시작된다.
- [ ] 진행률 퍼센트와 현재 작업 문구가 함께 보인다.
- [ ] 시스템 정보 수집과 가상화 점검이 모두 완료될 때까지 흐름이 멈춘 것처럼 보이지 않는다.
- [ ] 점검 완료 후 제목이 `시스템 점검 완료`로 바뀐다.
- [ ] 완료 안내 문구가 아래처럼 **2줄로 자연스럽게 표시**된다.
  - `최초 실행에 따른 하드웨어 및 가상화 설정 검사가 완료되었습니다.`
  - `아래 항목에 대한 조치 필요 여부를 확인하세요.`
- [ ] 완료 화면의 footer 버전 정보가 **스크롤 없이 바로 보인다**.
- [ ] 화면 하단에 `조치 시작` 버튼과 `닫기` 버튼이 표시된다.
- [ ] `닫기` 버튼을 클릭하면 앱이 종료된다.

### 조치 요약 표시
- [ ] 조치 필요 항목이 있으면 완료 화면 요약 카드에 **최대 4건**이 표시된다.
- [ ] 표시되는 요약은 현재 구현대로 **조치 필요 항목 중심**으로 보인다.
- [ ] 조치 필요 항목이 있으면 `조치 시작` 버튼이 활성화된다.
- [ ] 조치 필요 항목이 없으면 `조치가 필요한 항목이 없습니다.` 문구가 표시된다.
- [ ] 조치 필요 항목이 없으면 `조치 시작` 버튼이 비활성화된다.

### CSV 저장 표시
- [ ] 조치 필요 항목이 없을 때, 저장된 CSV 파일명이 완료 화면에 표시된다.
- [ ] 저장 실패 시 오류 문구가 표시된다.

---

## 5. 자동 저장 CSV 확인

앱 시작 자동 점검 후 생성되는 파일 기준입니다.

- [ ] 시스템 정보 CSV가 자동 저장된다.
- [ ] 가상화 점검 CSV가 자동 저장된다.
- [ ] 상태바 문구에 저장된 두 파일명이 반영된다.
- [ ] 두 CSV 모두 Excel 또는 메모장에서 정상적으로 열 수 있다.
- [ ] 파일명 규칙이 현재 구현과 일치한다.
- [ ] 저장 경로가 EXE 위치 기준 `logs/` 하위인지 확인한다.

---

## 6. 메인 메뉴 / 내비게이션

- [ ] 메인 메뉴에 아래 3개 작업이 모두 표시된다.
  - 시스템 사양 체크
  - 가상화 설정 점검
  - VBS 및 Hyper-V 비활성화
- [ ] 메뉴 진입 시 자동 점검 결과 배지가 이미 채워진 상태로 표시된다.
  - 시스템 사양 체크 → `완료` 배지
  - 가상화 설정 점검 → `N개 조치 필요` 또는 `정상` 배지
  - VBS 및 Hyper-V 비활성화 → 배지 없음
- [ ] 각 패널에서 뒤로 가기 시 메인 메뉴로 복귀한다.

---

## 7. 시스템 정보 패널

- [ ] `시스템 사양 체크` 진입 시 정상 로드된다.
- [ ] 수집 완료 후 총 수집 항목 수가 표시된다.
- [ ] 상세 항목은 UI에 직접 표시되지 않으며, CSV 내보내기로 확인한다.
- [ ] 수동 CSV 내보내기가 성공한다.
- [ ] 내보낸 CSV에서 이벤트 로그 요약의 한글 깨짐이 없다.
- [ ] `재수집` 버튼으로 시스템 정보를 다시 수집할 수 있다.

---

## 8. 가상화 점검 패널

### 기본 동작
- [ ] `가상화 설정 점검` 실행이 정상 완료된다.
- [ ] 상단 요약 카드가 아래 3개로 표시된다.
  - 조치 필요
  - 정상/비활성
  - 확인 불가
- [ ] `재점검`, `CSV 내보내기` 버튼이 정상 동작한다.

### 현재 구현 기준 표시 원칙
- [ ] 표에는 현재 구현대로 **조치 필요 항목**(`action_required`)과 `Windows Hello` 경고 항목(`whfb_check`)만 표시된다.
- [ ] 조치 필요 항목도 없고 WHfB도 없으면 `✅ 모든 항목이 VM 호환 상태입니다. 조치가 필요하지 않습니다.` 메시지가 표시된다.
- [ ] 자동 조치 제외 legacy 항목은 이 표에서 보이지 않으며, CSV에서 확인한다.
- [ ] Hyper-V / WSL / `hypervisorlaunchtype` 상태가 실제 시스템 상태와 일치한다.
- [ ] VBS / HVCI / Credential Guard / LSA 관련 항목 상태가 실제 시스템 상태와 크게 어긋나지 않는다.

### 액션 배너
- [ ] 조치가 필요한 그룹이 있으면 하단 배너에 `N개 작업 그룹에서 조치가 필요합니다`가 표시된다.
- [ ] 배너에서 `비활성화 실행 →` 버튼으로 비활성화 패널로 바로 이동할 수 있다.
- [ ] 조치가 필요 없으면 `모든 항목이 VM 호환 상태입니다`가 표시된다.

---

## 9. 누락값 / 미설정 정책

대상 레지스트리 값 일부를 의도적으로 삭제한 상태로 검증합니다.

- [ ] 점검 결과 상세 정보에 `<미설정>` 또는 동등한 표현으로 표시된다.
- [ ] 단순히 값이 없다는 이유만으로 자동 조치 필요로 잡히지 않는다.
- [ ] 비활성화 실행 시 로그에 `값 없음, 생성하지 않음` 성격의 메시지가 남는다.
- [ ] 재점검 시 Windows가 별도로 재생성하지 않은 이상 값이 없는 상태가 유지된다.

---

## 10. WHfB (Windows Hello for Business) 흐름

### WHfB 미감지 환경
- [ ] 시작 점검 완료 화면에 WHfB 경고 배너가 보이지 않는다.
- [ ] 조치 필요 항목이 있으면 `조치 시작` 버튼이 활성화된다.
- [ ] 비활성화 패널에 WHfB 경고 배너가 보이지 않는다.

### WHfB 감지 환경
- [ ] 시작 점검 완료 화면에 `⚠ Windows Hello for Business 감지됨` 경고가 표시된다.
- [ ] 경고 문구에 먼저 WHfB를 비활성화하라는 안내가 나온다.
- [ ] 시작 점검 완료 화면의 `조치 시작` 버튼이 비활성화된다.
- [ ] 비활성화 패널 상단에도 WHfB 경고 배너가 표시된다.
- [ ] 가상화 점검 표에서 WHfB 항목이 별도 행으로 표시된다 (info 색상 배지).
- [ ] WHfB 항목은 추가 선택형 레지스트리 조치 체크박스에 나타나지 않는다.

---

## 11. 비활성화 패널

- [ ] 비활성화 패널 상단 경고 박스가 현재 구현된 작업 범위(Hyper-V/WSL/VBS/코어 격리/bcdedit)를 정확히 설명한다.
- [ ] 점검을 수행한 경우, `점검 결과 기반 실행 예정 항목` 박스가 현재 그룹 선택 상태를 반영한다.
- [ ] 점검을 수행하지 않은 경우, `가상화 점검 없이 모든 항목을 일괄 처리합니다.` 안내가 표시된다.
- [ ] 추가 선택 가능한 legacy 레지스트리 후보가 있으면 `추가 레지스트리 조치 N개는 실행 전 확인 창에서 체크박스로 선택할 수 있습니다.` 안내가 표시된다.
- [ ] 로그 영역은 초기 상태에서 `비활성화 실행 버튼을 클릭하면 작업이 시작됩니다.` 대기 문구를 보여준다.

---

## 12. 비활성화 확인 모달 (warning 단계)

- [ ] `비활성화 실행` 클릭 시 전체화면 경고 모달이 열린다.
- [ ] 자동 조치 대상이 `조치 예정 항목` 영역에 표시된다.
- [ ] 추가 선택형 레지스트리 후보가 있으면 별도 체크박스 섹션이 표시된다.
- [ ] 체크박스 선택/해제 시 총 작업 수가 즉시 반영된다.
- [ ] 추가 선택 후보가 없으면 체크박스 섹션이 나타나지 않는다.
- [ ] 선택된 작업이 0개면 `조치 시작` 버튼이 비활성화된다.
- [ ] `취소` 버튼으로 모달을 닫을 수 있다.

### 현재 구현 기준 주의
- [ ] `InspectOnly` 항목은 체크박스 후보로 표시되지 않는다.
- [ ] WHfB 경고 항목은 체크박스 후보로 표시되지 않는다.

---

## 13. 비활성화 실행 (running 단계)

- [ ] 실행 시작 후 `조치 진행 중...` 화면으로 전환된다.
- [ ] 진행률 퍼센트가 표시된다.
- [ ] 현재 작업명이 실시간으로 갱신된다.
- [ ] 상태바와 비활성화 패널 로그 영역도 함께 갱신된다.
- [ ] Hyper-V, WSL, VBS, 코어 격리, 추가 선택 레지스트리 조치가 실제 선택 상태에 맞게만 실행된다.
- [ ] `Hyper-V 비활성화` 실행 후 로그에 `hypervisorlaunchtype off` 성공 메시지가 남는다.

---

## 14. 비활성화 결과 (complete 단계)

### 성공 / 부분 실패 공통
- [ ] 완료 후 결과 화면으로 전환된다.
- [ ] 로그 파일명과 레지스트리 백업 파일명이 있으면 표시된다.
- [ ] `지금 재부팅하시겠습니까?` 흐름이 정상 표시된다.

### 성공 케이스
- [ ] 성공 아이콘(녹색 체크)과 `조치 완료` 문구가 표시된다.

### 부분 실패 케이스
- [ ] 오류 아이콘(빨간 X)과 `일부 조치 실패` 문구가 표시된다.
- [ ] 실패 항목이 로그에 남는다.

### 재부팅 분기
- [ ] `예 — 지금 재부팅 (5초 후)` 선택 시 재부팅 예약이 동작한다.
- [ ] `아니요 — 나중에 재부팅` 선택 시 수동 재부팅 안내 화면으로 전환된다.
- [ ] 수동 재부팅 안내 화면에서 `닫기` 버튼으로 모달을 닫을 수 있다.

---

## 15. 운영 로그 / 백업 / 재점검

- [ ] 비활성화 후 운영 로그가 저장된다.
- [ ] VBS / 코어 격리 조치가 있었다면 레지스트리 백업이 저장된다.
- [ ] 재실행 또는 재점검 시 변경된 Hyper-V / WSL / VBS / HVCI 상태가 반영된다.
- [ ] 필요 시 추가 선택형 레지스트리 조치 결과도 재점검에 간접적으로 반영된다.

---

## 16. 레이아웃 / 고정 창 크기

현재 창 설정 기준:
- 기본 크기: `660 x 700`
- 최소 크기: `560 x 620`
- 크기 조절 불가, 최대화 불가

검증 항목:
- [ ] 시작 점검 화면에서 제목 / 설명 / 요약 / 경고 / 버튼 / footer가 한 화면 안에 자연스럽게 배치된다.
- [ ] 비활성화 패널에서 상단 안내, 실행 예정 항목, 로그 영역이 겹치지 않는다.
- [ ] 비활성화 확인 모달에서 체크박스 후보가 있어도 버튼 영역이 깨지지 않는다.
- [ ] 최소 지원 크기 기준에서도 주요 조작 버튼이 잘리지 않는다.

## 18. QA 기록 항목

테스트 결과에는 아래 정보를 함께 남깁니다.

- Windows 버전 / 빌드
- 테스트 장비 종류(실장비 / VM / 스냅샷 여부)
- 초기 활성화 상태
  - Hyper-V
  - WSL
  - VBS
