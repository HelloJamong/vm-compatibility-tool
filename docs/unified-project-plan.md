## 2. 프로젝트 한 줄 정의

이 프로젝트는 **Windows 환경에서 VM 호환성을 저해하는 가상화/보안 관련 설정을 점검하고, 필요한 경우 선택적으로 비활성화할 수 있게 돕는 관리자 권한 기반 데스크톱 도구**이다.

핵심 대상은 다음과 같다.

- Hyper-V
- WSL / VirtualMachinePlatform
- hypervisorlaunchtype
- VBS / HVCI / Credential Guard / LSA 관련 정책/레지스트리
- 코어 격리 관련 레지스트리
- Windows Hello for Business(경고/참고용)

## 3. 현재 기준 구현 스택

### Frontend
- Svelte 5
- TypeScript
- Vite 6

### Backend
- Rust
- Tauri v2
- WMI / Registry / PowerShell 기반 수집 및 조치

### 현재 런타임 기준
- **현재 저장소의 기준 구현은 Tauri 앱이다.**
- 과거 WPF/C# 구현은 현재 런타임 기준이 아니며, 새 기능은 **Rust + Svelte 기준으로만** 확장한다.

## 5. 현재 구현된 기능 정의

### 5.1 시스템 정보 수집
현재 수집 대상:
- OS 버전 / 빌드 / 에디션
- CPU 정보 및 하드웨어 가상화 여부
- 메모리 / 디스크 / 부팅 정보
- 메인보드 / GPU / 전원 계획
- 최근 이벤트 로그 요약

현재 상태:
- 앱 시작 시 자동 수집 수행
- 시스템 정보 패널에서 수집 완료 상태 확인 가능
- 현재 상세 내용은 **수동 CSV export**로 확인하는 구조

### 5.2 가상화 호환성 점검
현재 점검 대상:
- Hyper-V 기능 상태
- WSL / VirtualMachinePlatform 상태
- `hypervisorlaunchtype`
- VBS / HVCI / Credential Guard / LSA 관련 레지스트리
- legacy / reference-only 항목
- WHfB 감지 결과

현재 UI 정책:
- 사용자에게는 **조치 필요 항목 중심으로 표시**
- 전체 결과는 CSV로 확인 가능
- WHfB는 `manifest_id = "whfb_check"` 로 구분되는 **경고 전용 항목**

### 5.3 선택적 비활성화
현재 비활성화 가능 대상:
- Hyper-V 비활성화
- WSL 비활성화
- VBS 레지스트리 비활성화
- 코어 격리 레지스트리 비활성화
- `hypervisorlaunchtype off`
- 재부팅 요청

현재 동작 원칙:
- 점검 결과가 있으면 **필요한 그룹만 선택 실행**
- 점검 없이 실행하면 전체 그룹 일괄 실행
- 실행 결과는 UI 로그에 누적 표시

### 5.4 자동 저장 기능
현재 완료된 자동 저장:
- 비활성화 실행 후 운영 로그 자동 저장
- 비활성화 실행 전 레지스트리 백업 파일 자동 저장

현재 미완료 자동 저장:
- 시스템 정보 CSV 자동 저장

## 6. 레지스트리 / 정책 정의

### 6.1 점검 정책
- 값이 없으면 **미설정**으로 표시한다.
- 값이 없다고 해서 점검 과정에서 새 값을 만들지 않는다.

### 6.2 비활성화 정책
- 값이 실제로 존재하고 활성 상태일 때만 목표값(`0`)으로 변경한다.
- 값이 없으면 **생성하지 않고 skip** 한다.
- 이미 비활성 상태면 그대로 유지하고 로그에 남긴다.

### 6.3 manifest 정책
레지스트리 제어는 `registry_manifest.rs`를 source of truth로 사용한다.

분류:
- `DisableWrite`
- `InspectOnly`
- `ExcludedLegacy`

원칙:
- 점검과 조치가 같은 기준을 공유해야 한다.
- 표시용 legacy 항목은 자동 조치에 포함하지 않는다.

## 7. 현재 UI/UX 기준

### 메인 방향
- 복잡한 정보를 모두 한 화면에 노출하지 않는다.
- 사용자에게는 **즉시 필요한 진단/조치 정보**가 먼저 보여야 한다.
- 운영자 관점에서 빠르게 판단 가능한 밀도 높은 UI를 유지한다.

### 현재 구현 기준
- 앱 시작 시 자동 점검 수행
- 메인 메뉴는 3개 주요 액션 중심
- 가상화 점검 화면은 조치 필요 항목 우선
- 비활성화 화면은 실제 실행 예정 그룹을 요약해서 보여줌
- 버전/상태/진행도는 하단 상태영역에서 유지

### 아직 남은 UI 작업
- QA 결과에 따른 문구 정리
- warning / badge / row wording polish
- 필요 시 최소 크기에서의 가독성 보완

## 8. 현재 남은 핵심 작업

## P0 — 가장 먼저 진행할 작업

### 8.1 Stage 4: 시스템 정보 CSV 자동 저장
목표:
- 현재 수동 내보내기 기반인 시스템 정보 export를 **자동 저장 방식으로 전환**한다.

의도:
- 운영자가 별도 저장 버튼을 누르지 않아도 결과 확보 가능
- 비활성화 로그/백업 저장 흐름과 일관성 확보

작업 정의:
- 시스템 정보 수집 완료 시 `{exe_dir}/logs/` 하위에 자동 저장
- 파일명 규칙 정의 필요
  - 예: `YYYYMMDD_HHMMSS_{COMPUTERNAME}_system_info.csv`
- 현재 수동 CSV 버튼은 제거하거나, 자동 저장 이후 보조 기능으로 재정의할지 결정 필요

권장 방향:
- **기본은 자동 저장**
- 수동 export 버튼은 Stage 4 완료 시 제거하는 쪽을 우선 고려

완료 조건:
- 시스템 정보 수집 직후 CSV가 자동 생성됨
- Excel/메모장에서 한글 깨짐 없이 열림
- 상태 메시지 또는 UI 로그에서 저장 위치 확인 가능

### 8.2 Windows 실기 QA
기준 문서:
- `docs/windows-manual-qa-checklist.md`

필수 확인 항목:
- 관리자 권한 실행
- Hyper-V / WSL / BCD 상태 실제 반영
- missing-key 정책
- selective disable 정책
- WHfB 경고 노출
- 비활성화 후 로그/백업 파일 생성
- 재부팅 요청 동작

완료 조건:
- 체크리스트 주요 항목 완료
- 발견 이슈를 이 문서 또는 후속 QA 문서에 기록
